# 포트폴리오 콘텐츠 작성 가이드

이 스킬은 `src/content/` 하위의 프로젝트(projects), 경력(careers) index.mdx 파일을 작성하거나 수정할 때 반드시 따라야 하는 가이드입니다.

## 절대 규칙

1. **사실만 작성한다.** meta.tsx, GitHub README, 실제 코드, 사용자가 제공한 정보만 기반으로 작성합니다. 추측하거나 지어낸 내용을 포함하지 않습니다. 모르는 부분은 비워두거나 사용자에게 확인합니다.
2. **수치를 조작하지 않는다.** 다운로드 수, 사용자 수, 수상 실적 등은 검증된 출처에서만 인용합니다. 대략적인 수치도 "약", "이상" 등으로 명시합니다.
3. **존재하지 않는 기능이나 성과를 만들어내지 않는다.** 실제로 구현/달성하지 않은 것을 기술하지 않습니다.

## 톤 & 문체

- **~했습니다** 체를 사용합니다.
  - 올바른 예: "React Native 기반으로 모바일 앱을 개발했습니다."
  - 잘못된 예: "모바일 앱을 개발했다." (반말)
  - 잘못된 예: "모바일 앱을 개발하게 되었습니다." (피동/회피)
- 간결하고 명확한 문장을 사용합니다. 한 문장이 두 줄을 넘기지 않습니다.
- 마크다운 볼드(**), 이모지를 남용하지 않습니다.
- 기술 용어는 그대로 사용합니다 (React Native, Kubernetes 등 번역 불필요).

## 필수 포함 요소

모든 프로젝트/경력 콘텐츠는 아래 세 가지를 반드시 담아야 합니다:

### 1. WHY — 왜 했는가

- 어떤 문제가 있었는지, 왜 이 프로젝트를 시작했는지
- 기존 방식의 한계나 불편함이 무엇이었는지
- 예: "기존에는 수동으로 관리하던 예약 프로세스에서 오류가 잦아, 이를 자동화하기 위해 시작했습니다."

### 2. WHAT — 구체적으로 무엇을 했는가

- 본인이 실제로 맡은 역할과 수행한 작업을 구체적으로 기술
- 기술 스택은 나열이 아니라 "무엇을 위해 어떤 기술을 썼는지" 맥락과 함께 설명
- 예: "NestJS로 REST API 서버를 구축하고, Prisma ORM으로 PostgreSQL과 연동했습니다."
- 예 (나쁜 예): "NestJS, Prisma, PostgreSQL 사용" (맥락 없는 나열)

### 3. IMPACT — 어떤 결과/임팩트가 있었는가

- 정량적 성과가 있다면 수치로 표현 (다운로드 수, 사용자 수, 시간 단축률, 비용 절감 등)
- 정성적 성과라도 구체적으로 (수상 실적, 서비스 출시, 실사용 규모 등)
- 성과가 명확하지 않으면 학습한 점이나 기술적 도전 포인트를 대신 기술
- 예: "npm에 배포하여 누적 300회 이상 다운로드를 달성했습니다."
- 예: "이 프로젝트를 통해 Kubernetes의 Pod 스케줄링과 리소스 관리를 실무 수준으로 익혔습니다."

## 구조 템플릿

```mdx
{/* 이미지 import가 있다면 상단에 배치 */}
import Image from 'next/image';
import img from './assets/content-1.png';

{/* 첫 단락: 프로젝트 한 줄 요약 + WHY */}
OOO을 위해 OOO를 개발했습니다. (왜 시작했는지 1~2문장)

{/* 이미지가 있다면 여기 배치 */}
<Image src={img} alt="" />

## 배경

(WHY를 더 구체적으로. 기존 문제점, 동기 등)

## 주요 작업

(WHAT을 구체적으로. 본인이 한 일 중심)

## 성과

(IMPACT. 정량/정성적 결과)
```

## 프로젝트 vs 경력 차이

- **프로젝트 (projects/)**: 기술적 내용 중심. 작동 방식, 아키텍처, 기술적 도전 포인트를 상세히 기술합니다.
- **경력 (careers/)**: 역할과 기여 중심. 회사/기관에서 본인이 맡은 역할, 수행한 업무, 성과를 기술합니다. 회사 전체 소개보다는 본인의 기여에 초점을 맞춥니다.

## 에셋(이미지) 규칙

- 이미지는 `./assets/` 폴더에서 import합니다.
- 존재하지 않는 이미지를 import하지 않습니다. 반드시 `ls assets/`로 확인 후 사용합니다.
- 이미지 레이아웃 컴포넌트: `<ImageGrid cols={2|3|4}>`, `<PhoneScreens>`, `<SideBySide>`
- alt 텍스트는 빈 문자열("")이어도 괜찮습니다.
